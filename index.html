Guia Futurista: Do Local ao GitHub

Bem-vindo ao futuro do controle de versão. Este guia detalhado mostra como levar seu projeto do seu ambiente de desenvolvimento local para um repositório no GitHub, tudo através da linha de comando (CLI). Uma habilidade essencial para o desenvolvedor moderno que busca eficiência e automação.

Pré-requisitos Essenciais

  • Conta no GitHub: Essencial. Se não tiver, crie uma em github.com.
  • Git Instalado: O sistema de controle de versão. Verifique com git --version. Se não tiver, baixe aqui.
  • GitHub CLI Instalada: A ferramenta que conecta seu terminal ao GitHub. Verifique com gh --version. Se não tiver, instale aqui.

O Plano de Voo: Passo a Passo

Passo 1: Navegue até a Raiz do seu Projeto

Abra seu terminal e use o comando cd (change directory) para entrar na pasta principal do seu projeto.

cd /caminho/completo/para/seu/projeto
Passo 2: Inicie o Controle de Versão com Git

Se o seu projeto ainda não é um repositório Git, este comando cria uma nova "sala de controle" (uma pasta oculta .git) para rastrear todas as suas alterações.

git init
Passo 3: Prepare e Registre sua Versão Inicial

Primeiro, adicione todos os arquivos ao "stage" (área de preparação) com git add .. Em seguida, "tire uma foto" do estado atual do seu projeto com git commit, adicionando uma mensagem que descreva essa versão.

git add .
git commit -m "Commit inicial: Estrutura do projeto"
Passo 4: Autentique-se no GitHub via CLI

Este é um passo único. Você vai conectar seu terminal à sua conta do GitHub de forma segura. O terminal irá te guiar para autenticar através do navegador.

gh auth login
Passo 5: Crie o Repositório Remoto e Envie seu Código

A mágica acontece aqui. O comando gh repo create vai criar um novo repositório no seu GitHub e, com as flags adicionais, já vai configurar o link remoto e enviar (push) seu código para lá.

gh repo create nome-do-seu-repositorio --public --source=. --remote=origin --push
  • Substitua nome-do-seu-repositorio pelo nome que desejar.
  • Use --private em vez de --public para um repositório privado.

Missão Cumprida! E Agora?

Seu código agora vive no GitHub! Para futuras atualizações, o processo é mais simples:

  1. Faça suas alterações no código.
  2. Adicione as alterações: git add .
  3. Faça o commit: git commit -m "Descreva suas alterações"
  4. Envie para o GitHub: git push
